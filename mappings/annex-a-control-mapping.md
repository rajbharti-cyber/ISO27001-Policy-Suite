# 📋 ISO 27001 Annex A Control Mapping

## 📌 Overview

This document demonstrates a sample mapping between ISO 27001 Annex A security controls and implemented governance policies within the policy suite.

The purpose of this mapping is to:
- Support audit readiness
- Improve traceability
- Demonstrate compliance alignment
- Strengthen governance visibility
- Validate policy coverage against ISO requirements

---

# 🎯 Mapping Objectives

- Align governance policies with ISO 27001 controls
- Identify control ownership and policy coverage
- Support compliance and audit activities
- Improve ISMS documentation maturity

---

# 📊 Annex A Control Mapping

| ISO 27001 Annex A Control | Policy Coverage |
|---|---|
| A.5 Information Security Policies | Information Security Policy |
| A.5.15 Access Control | Access Control Policy |
| A.5.23 Information Security for Use of Cloud Services | Network Security Policy |
| A.5.24 Incident Management Planning | Incident Response Policy |
| A.5.30 ICT Readiness for Business Continuity | BCP/DR Policy |
| A.5.9 Inventory of Information Assets | Asset Management Policy |
| A.5.19 Supplier Relationships | Supplier Security Policy |
| A.8.24 Use of Cryptography | Cryptographic Controls Policy |
| A.8.15 Logging | Logging & Monitoring Policy |
| A.8.16 Monitoring Activities | Logging & Monitoring Policy |
